## Abstract 

Memory systems are crucial to personalized LLMs by mitigating the context window limita-tion in capturing long-term user-LLM conversa-tions. Typically, such systems leverage multi-ple agents to handle multi-granular memory con-struction and personalized memory retrieval tasks. To optimize the system, existing methods focus on specializing agents on their local tasks inde-pendently via prompt engineering or fine-tuning. However, they overlook cross-agent collaboration, where independent optimization on local agents hardly guarantees the global system performance. To address this issue, we propose a Collaborative Reinforcement Learning Framework for Multi-Agent Memory Systems (CoMAM), jointly op-timizing local agents to facilitate collaboration. Specifically, we regularize agents’ execution as a sequential Markov decision process (MDP) to embed inter-agent dependencies into the state transition, yielding both local task rewards ( e.g., 

information coverage for memory construction) and global rewards ( i.e., query-answer accuracy). Then, we quantify each agent’s contribution via group-level ranking consistency between local and global rewards, treating them as adaptive weights to assign global credit and integrate local-global rewards. Each agent is optimized by these integrated rewards, aligning local improvements with the global performance. Experiments show CoMAM outperforms leading memory systems, validating the efficacy of our proposed collabora-tive reinforcement learning for joint optimization.

To be more specific, if the fine-grained construction agent is optimized by treating retrieval agents as fixed environment components, it may store redundant or noisy content that im-pairs retrieval effectiveness. This moti-vates us to explore jointly optimizing multiple agents in the memory system to facilitate collaboration, which faces two challenges (see Figure 2): • Heterogeneity and Asynchrony: Multi-granular con-struction and retrieval agents operate with heterogeneous configurations and asynchronous executions across dif-ferent tasks. Their discrete parameter space and inherent structural complexity complicate end-to-end optimization. • Local-Global Alignment: Fostering collaboration re-quires aligning local improvement with the global perfor-mance. However, directly assigning equal shares of the global objective to individual agents introduces ambiguity in task execution, as agents’ contributions are divergent. To address these challenges, we introduce CoMAM, a Collaborative reinforcement learning framework for Multi-Agent Memory systems. Specifically, we formulate a stan-dard memory system with three specialized agents for multi-granular memory construction and retrieval: an extraction agent to construct fine-grained memories ( e.g., key events and facts), a profile agent to construct coarse-grained mem-ories ( e.g., users’ personalized preferences), and a retrieval agent to retrieve memories and answer queries. To optimize these heterogeneous agents jointly, we frame their asyn-chronous execution as a sequential Markov decision process (MDP), which embeds inter-agent dependencies into state transitions and yields both local task-specific rewards ( e.g., 

information coverage for memory construction) and global system rewards ( i.e., query answer accuracy). To achieve local-global alignment, CoMAM quantifies each agent’s sys-tem contribution by measuring group-level ranking consis-tency between its local and global rewards, then adaptively allocates global rewards to agents based on this consistency. Each agent is ultimately optimized with a combination of its task-specific reward and adaptively assigned global re-ward share, driving end-to-end collaboration toward the global objective. Extensive experiments on long-term per-sonalized conversational tasks show CoMAM outperforms leading memory baselines, validating the efficacy of our collaborative reinforcement learning for jointly optimizing multi-agent memory systems. 

## 2. Related work 

Memory System for Personalized LLMs Beyond de-livering general knowledge, personalized LLMs generate personalized responses to users (Chen et al., 2024a; Zhang et al., 2025), such as recalling past conversational experi-ences or producing preference-aligned suggestions (Jiang et al., 2025), based on user-LLM conversational histories. To address LLMs’ context window limitation on long-term conversational data, recent work adopts memory systems (Hu et al., 2025; Li et al., 2025; Chhikara et al., 2025; Yan et al., 2025b) to manage user-specific conversational histo-ries, which are composed of multiple agents to handle mem-ory construction and retrieval subtasks separately. For ex-ample, MIRIX (Wang & Chen, 2025) designs a meta agent and six specialized agents to manage memory. Mem −α

(Wang et al., 2025) and Mem1 (Zhou et al., 2026) lever-age reinforcement learning (RL) to optimize the memory construction and memory retrieval agents, respectively. 

Optimization of Multi-agent System Multi-agent Sys-tems (MAS) (Sun et al., 2024; Chen et al., 2024b) lever-age multiple specialized agents with heterogeneous con-figurations and asynchronous execution to tackle complex tasks beyond the capacity of a single agent. To boost MAS performance, existing methods focus on independent op-timization, primarily via prompt engineering (Patil et al., 2024; Wu et al., 2024) or fine-tuning of agents on specific tasks (Subramaniam et al., 2025; Yan et al., 2025b). Yet such methods yield only marginal gains, as they overlook the value of joint optimization for inter-agent collaboration. Consequently, recent research has shifted to reinforcement learning (RL)-based collaborative training for end-to-end system optimization, such as MAPoRL (Park et al., 2025) and MARFT (Liao et al., 2025). 

Credit Assignment Credit assignment(Zhou et al., 2020; Bo et al., 2024a; Tseng et al., 2022) concerns properly at-tributing a team’s global outcome to the specific agents in Multi-Agent Reinforcement Learning (MARL). This is par-ticularly difficult in quantifying agents’ contributions to the outcome, which is often obscured by the actions of other agents and environmental noise. Naive credit assignment approaches evenly distribute global team rewards across agents (Zhao et al., 2026; Hong et al., 2025). Though sim-ple, this leads to significant credit assignment ambiguity: agents cannot distinguish their specific impact, resulting in high variance in policy updates and unstable convergence. 2Collaborative Multi-Agent Optimization for Personalized Memory System

We begin by defining the task-specific agents alongside the local and global rewards in Sec 4.1. ## 6. Conclusion 

In this paper, we propose CoMAM, a collaborative rein-forcement learning framework for jointly optimizing agents in personalized memory systems. It tackles the issue that in-dependent agent optimization can not ensure global system performance without cross-agent collaboration. To enable joint optimization, we regularize the asynchronous execu-tions of heterogeneous agents as a sequential MDP, which embeds inter-agent dependencies into state transitions and yields both local and global rewards. To align local improve-ment with global performance, we introduce an adaptive credit assignment mechanism that allocates global rewards to individual agents based on group-level ranking consis-tency between global and local rewards. Experiments on long-term personalized conversational benchmarks demon-strate that CoMAM outperforms leading memory baselines, validating the effectiveness of our collaborative RL frame-work. ## B. Details of Experimental Settings 

B.1. Dataset 

In the PersonaMem benchmark, each interaction history consists of 10, 20, or 60 multi-turn sessions (corresponding to 32k, 128k, or 1M tokens) to form varying context lengths. Each session contains 15–30 conversation turns, designed to naturally reveal or update user preferences. In addition, each history is paired with a suite of subsequent queries across seven types, enabling comprehensive evaluation of personalized response capabilities. For PersonaMem’s three context length settings (32k, 128k, and 1M tokens), we partition their training and validation sets based on these queries. The statistical breakdown of training and validation queries is summarized in Table 6. To process the training set, we first identify the specific session within the full conversational history that contains explicit evidence relevant to each query q, and we define this session as the query’s targeted conversational history H. We then construct (H, q ) data pairs required for generating individual MDP trajectories as the core training data. The position of each target session within the complete interaction history is specified in the original benchmark. Ground-truth evidence Eq

for each query q is extracted using the Seed-1.6 model and validated through human evaluation. For evaluation, multiple distinct queries may share the same full conversational history that includes multiple sessions. B.2. Detailed Baselines 

Here, we detailed our baseline methods, including methods without memory, prompt-based memory systems, and RL-optimized memory systems. 

Methods without memory: 

• Base denotes the direct injection of raw conversational data as input to large language models (LLMs) for query answering tasks. Specifically, when the length of the input conversation exceeds the context window constraint, the input is truncated at the context window limit. • RAG (Lewis et al., 2020) answers queries by retrieving raw conversational history via embedding-based similarity matching. The embedding model employed is all-MiniLM-L6-v2, and ultra-long input contexts are uniformly partitioned into fixed-length segments, each consisting of 2,048 tokens. 

Prompt-based memory systems 

• A-Mem (Xu et al., 2025) instantiates a memory system through multiple agents with dedicated prompts, including the memory retrieval construction, precursor memory evolution, and retrieval agents; • CAM (Li et al., 2025) builds a hierarchical memory system via multiple agents, where dedicated agents undertake node construction (via ego-centric disentanglement and node replication) and information aggregation (for hierarchical abstraction). We implement it on the PersonaMem benchmark. • MemoryBank (Zhong et al., 2024) construct a human-like memory for LLMs that uses a forgetting curve to remember and forget information over time. Similarly, we implement it on the PersonaMem benchmark. 

RL-based memory: 

• Mem1 (Zhou et al., 2026) proposes a reinforcement learning (RL) framework that empowers agents to conduct efficient multi-turn information retrieval and reasoning for query answering. The optimized agents prioritize direct memory retrieval from history, rather than memory construction. • Memory-R1 (Yan et al., 2025b) constructs a Memory Manager that learns optimal memory operations (ADD, UPDATE, DELETE) and an Answer Agent tasked with distilling relevant information from retrieved entries. Uniquely, the system is optimized via independent reinforcement learning (RL) in two stages: the Memory Manager is optimized first, followed by the Answer Agent.

## C. More Detailed Experimental Results

## D. Discussion 

D.1. Difference with existing memory framework 

Existing memory systems and our CoMAM differ fundamentally in optimization paradigm and reward design: • Optimization Paradigm: Independent Optimization vs. Joint Optimization Existing frameworks primarily adopt independent optimization. For example, MIRIX (Wang & Chen, 2025) designs a meta-agent and six specialized agents for memory management, optimizing each via tailored prompts; Memagent (Yu et al., 2026) focuses on a single retrieval agent, training it with query-answer accuracy as the reward; Memory-R1 (Yan et al., 2025b) splits the system into a Memory Manager and Answer Agent, training them in separate stages—first optimizing the Manager with the Agent frozen, then freezing the Manager to optimize the Agent. This approach ignores inter-agent collaboration: memory construction is not adjusted based on retrieval needs, and the Agent can only adapt to a fixed Manager’s output, leading to “local optimal but global suboptimal” outcomes. CoMAM proposes end-to-end joint optimization via MDP regularization. We model the asynchronous execution of Extraction, Profile, and Retrieval Agents as a sequential MDP trajectory, where each agent’s output directly serves as the next agent’s input. All three agents are updated simultaneously during training, with inter-agent dependencies embedded into state transitions, ensuring they evolve collaboratively toward the global goal of query-answer accuracy. • Reward Design: Single Global Reward vs. Adaptive Credit Assignment Existing frameworks that optimize memory agents with RL primarily rely solely on final query-answer accuracy as the reward for all agents. For example, Mem-α and Mem1 (Zhou et al., 2026) use this single reward to train construction and retrieval agents respectively. Memory-R1 (Yan et al., 2025b) and GAM (Yan et al., 2025a) also adopt this design, using only answer accuracy to optimize their agents separately. This ambiguous credit assignment prevents agents from distinguishing their individual contributions, leading to unstable convergence. CoMAM introduces adaptive integration of local and global rewards. We design task-specific local rewards (e.g., Extraction Agent’s information coverage, Profile Agent’s abstraction rationality, Retrieval Agent’s retrieval precision) and combine them with the global reward (answer accuracy). Using NDCG to quantify the ranking consistency between each 14 Collaborative Multi-Agent Optimization for Personalized Memory System 

agent’s local rewards and the global reward, we dynamically allocate global credit based on this consistency. This ensures agents are rewarded both for excelling at their local tasks and contributing to global performance, avoiding ambiguous optimization. 

D.2. Difference with existing multi-agent system RL 

• Heterogeneous Policy vs. Homogeneous Policy : Some multi-agent reinforcement learning (RL) methods adopt a shared policy across all agents. For instance, CORY (Ma et al., 2024) proposes an RL framework featuring “Pioneer” and “Observer” agents that share a single policy model. These agents co-evolve through multi-agent collaboration and periodic role-swapping. In contrast, considering the heterogeneity and asynchrony of a memory system, CoMAM employs heterogeneous policies tailored to distinct agents. • Workflow: Sequential vs. Concurrent : Existing multi-agent system (MAS) methods often adopt a debate-style pipeline for reasoning tasks, such as math tasks. For instance, MAPoRL (Park et al., 2025) models agent interactions as concurrent debate, where multiple agents generate responses in parallel, focusing on persuasive collaboration for general reasoning. In contrast, CoMAM explicitly formulates agents’ executions as a sequential Markov Decision Process (MDP), embedding inter-agent dependencies into state transitions, ensuring that memory construction quality dynamically adapts to retrieval requirements and vice versa. • Collaboration Paradigm: Joint Optimization vs. Independent Alignment : Existing methods, such as Optimas (Wu et al., 2026), foster collaboration by aligning local rewards with global objectives via heuristic reward shaping for compound AI systems. However, it optimizes individual agents separately, leveraging locally-global aligned rewards rather than through end-to-end joint training. In contrast, CoMAM adopts a joint optimization paradigm: it regularizes the asynchronous execution of heterogeneous agents into a sequential MDP trajectory and updates all agents simultaneously. • Credit Assignment: Adaptive Assignment vs. Equal Allocation : Most existing systems use static credit assignment mechanisms without quantifying agents’ varying contributions: AT-GRPO (Zhao et al., 2026) distributes rewards equally across agents using fixed weights, ignoring task differences; CoMAS (Xue et al., 2026) adopts a zero-sum adversarial reward scheme that pits agents against each other, which is misaligned with the cooperative nature of memory systems; In contrast, CoMAM’s credit assignment is grounded in group-level ranking consistency (NDCG) between each agent’s local task rewards ( e.g., coverage for extraction, rationality for profiling, precision for retrieval) and the global system reward (query-answer accuracy). This design quantifies each agent’s contribution to the global system. It dynamically adjusts weights according to agent performance, ensuring adaptive credit allocation that aligns local improvements with global performance. 

## E. Limitation 

While CoMAM enables effective joint optimization for multi-agent memory systems, it is worth noting that our current implementation focuses on verifying the core logic of collaborative RL, with certain extended capabilities reserved for future exploration to better adapt to more complex scenarios. 

1. Limited Agent Task Scope: 

The framework currently leverages a compact agent set (Extraction, Profile, Retrieval) to focus on verifying the efficacy of collaborative RL for personalized memory tasks. To prioritize the core verification goal, we reserve extended agent capabilities (such as memory modification for consistency maintenance and redundant memory deletion) for future work. 

2. Static MDP Regularization for Training: 

Current MDP regularization models agent execution as a static sequential trajectory, enabling joint training of heterogeneous and asynchronous agents. Compared with the existing independent optimization training paradigm, this design can better enhance the collaborative capabilities of agents beyond task-specific capabilities. While effective in boosting inter-agent collaboration, we can further consider the dynamic deployment of agents in real-world scenarios, such as dynamic invocation frequencies and real-time memory updates. Future work will integrate these real-world dynamics into the joint training process to improve the consistency between training and deployment.
