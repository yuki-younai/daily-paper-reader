Abstract 

Multimodal web agents that process both screenshots and accessibility trees are increas-ingly deployed to interact with web interfaces, yet their dual-stream architecture opens an un-derexplored attack surface: an adversary who injects content into the webpage DOM simul-taneously corrupts both observation channels with a consistent deceptive narrative. Our vul-nerability analysis on MiniWob++ reveals that attacks including a visual component far out-perform text-only injections, exposing criti-cal gaps in text-centric VLM safety training. Motivated by this finding, we propose Dual-Modality Multi-Stage Adversarial Safety Train-ing (DMAST) , a framework that formalizes the agentŌĆōattacker interaction as a two-player zero-sum Markov game and co-trains both play-ers through a three-stage pipeline: (1) imi-tation learning from a strong teacher model, (2) oracle-guided supervised fine-tuning that uses a novel zero-acknowledgment strategy to instill task-focused reasoning under adversar-ial noise, and (3) adversarial reinforcement learning via Group Relative Policy Optimiza-tion (GRPO) self-play. On out-of-distribution tasks, DMAST substantially mitigates adver-sarial risks while simultaneously doubling task completion efficiency. Our approach signifi-cantly outperforms established training-based and prompt-based defenses, demonstrating gen-uine co-evolutionary progress and robust gen-eralization to complex, unseen environments. 

1 Introduction 

Web agents powered by Vision-Language Models (VLMs) are increasingly deployed to autonomously navigate and interact with web interfaces on behalf of users. Modern agents ingest both a rendered screenshot and a structured accessibility tree (AX-Tree) extracted from the DOM, leveraging com-plementary strengths: the screenshot captures rich visual context, while the accessibility tree provides precise labels and types of interactive elements. This dual-stream architecture has driven steady progress on web interaction benchmarks (Liu et al., 2018; Koh et al., 2024), yet it also opens an under-explored attack surface. An adversary who injects content into the webpageŌĆÖs DOM simultaneously corrupts both channels, presenting the agent with a consistent deceptive narrative that is far harder to detect than a single-modality perturbation. Despite growing concerns about agent secu-rity (Greshake et al., 2023; Xu et al., 2024), most existing work studies text-based prompt injections and image-based attacks in isolation, leaving a criti-cal gap: how vulnerable are multimodal web agents when attackers coordinate across modalities? We address this question with a systematic vulnerabil-ity analysis on the MiniWob++ benchmark using a state-of-the-art instruction-tuned VLM. The re-sults reveal that attacks including a visual compo-nent (either image-only (34.4% attacker success) or coordinated dual-modality (35.7%)) are much more effective at inducing safety jailbreaks than text-only injections (24.1%). This disparity aligns with recent findings that image-based injections of-ten bypass text-centric safety filters (Yeo and Choi, 2025), and it demonstrates that current VLMs are not equipped to handle visual deceptions such as typographic overlays, fake system dialogs, or phish-ing forms embedded in screenshots. Motivated by these findings, we propose Dual-Modality Multi-Stage Adversarial Safety Training (DMAST) , a framework for hardening multimodal web agents through adversarial co-evolution. We formalize the agentŌĆōattacker interaction as a two-player zero-sum Markov game and design a unified HTML injection mechanism where the attacker generates structured DOM modifications executed in the browser runtime. Both the agent and attacker are in-stantiated from the same VLM with shared weights, enabling efficient co-evolution: as the model learns to craft attacks, it simultaneously develops aware-ness of those patterns, which transfers to defense. Training proceeds through a three-stage pipeline. 

Stage 1 (Imitation Learning) distills expert trajec-tories from a stronger teacher model into a smaller student model, providing a stable initialization for both roles. Stage 2 (Oracle-Guided SFT) augments the training data with a novel denoising strategy: an oracle with privileged access to both clean and attacked views generates task-focused Chain-of-Thought reasoning that teaches the student to main-tain goal-directed behavior amid adversarial noise, without ever acknowledging the attack. Stage 3 (Adversarial RL) co-evolves the agent and attacker through self-play using Group Relative Policy Op-timization (GRPO), driving both players toward increasingly sophisticated strategies. Experiments on held-out MiniWob++ tasks and out-of-distribution VisualWebArena tasks demon-strate the effectiveness of DMAST. On our cu-rated VisualWebArena tasks with adversarial setup, the full pipeline reduces attacker success from 41.2% (base model) to 21.4% while improving agent task completion from 6.2% to 10.2%, out-performing baselines including SPAG (Chen et al., 2024), Automatic Red Teaming (Shi et al., 2025; Samvelyan et al., 2024), Online SFT (Zelikman et al., 2022), and pure prompt-based defenses. Each training stage provides cumulative improvements, with oracle-guided SFT instilling structural priors that persist through RL, and cross-checkpoint eval-uation confirms that both agent and attacker capa-bilities genuinely co-evolve through self-play. 

2 Related Work 

Safety of LLM-Based Web Agents. Indirect prompt injection attacks can compromise LLM-integrated applications by embedding malicious instructions in external data (Greshake et al., 2023). In the web agent context, AdvWeb (Xu et al., 2024) injects adversarial prompts into HTML fields in-visible to the rendered page. Benchmarks such as AgentDojo (Debenedetti et al., 2024) and Multi-Trust (Zhang et al., 2024) consistently reveal that current agents remain vulnerable to diverse safety attacks. 

Attacks on Multimodal Models. The integra-tion of visual encoders into LLMs expands the attack surface. Typographic attacks (Wang et al., 2025b; Qraitem et al., 2024; Chung et al., 2024) superimpose misleading text onto images, while 

adversarial perturbations (Shayegani et al., 2023; Aichberger et al., 2025) add imperceptible noise to fool image encoders. Beyond single-modality attacks, cross-modal injections (Wang et al., 2025a) simultaneously corrupt both visual and textual channels, exploiting the fusion process to hijack agent decision-making process and execute unau-thorized tasks. Yeo and Choi (2025) systematically evaluated eight modern multimodal models and found that image-based injections were among the most effective attack types, often bypassing text-only content filters. 

Adversarial Training and Self-Play. Adversar-ial reinforcement learning, where an agent co-evolves against a learned adversary, has proven 2effective in various domains including strategic games (Silver et al., 2017; Vinyals et al., 2019), dynamic control (Wurman et al., 2022), and se-curity (Cui et al., 2023). For LLMs, self-play has been applied to instruction following (Chen et al., 2024) and preference alignment (Wu et al., 2024). In multimodal safety, Rashid and Rivas (2024) and Schlarmann et al. (2024) used adver-sarial fine-tuning for robust vision encoders, and ProEAT (Lu et al., 2025) proposed a two-stage ad-versarial framework for jailbreak defense across modalities. Our work extends this paradigm to web agents facing coordinated multimodal attacks. 

3 Problem Formulation 

In this section, we formalize the multimodal web agent setting as a two-player game, describe our HTML-based injection mechanism, and present empirical evidence demonstrating the increased vulnerability of multimodal agents under visual attacks, justifying the need for our multimodal ad-versarial training framework. We focus on sensitive data leakage as the primary threat: the attacker aims to trick the agent into revealing private information (e.g., passwords) through crafted injections into the agentŌĆÖs observations.

3.1.1 HTML-Based Injection Mechanism 

We implement a unified HTML injection mech-anism: the attacker generates structured HTM-L/CSS code injected into the live webpage DOM via JavaScript execution. This design achieves (1) 

cross-modal consistency : the screenshot and acces-sibility tree always agree, (2) computational effi-ciency : requiring only JavaScript execution rather than GPU-based image generation, and (3) seman-tic richness : the attacker can create interactive ele-ments (forms, buttons) that appear in the accessibil-ity tree with proper labels, indistinguishable from legitimate elements. Concretely, the attackerŌĆÖs action ╬▒t =(target , position , html , css ) consists of a CSS selector specifying the target injection point, a po-sitional directive (prepend/append/before/after), an HTML payload, and optional CSS styling. This tuple is compiled into JavaScript, executed in the browser, and all injected elements are tagged for clean rollback between steps (details in Ap-pendix A). 4.1 Stage 1: Imitation Learning 

Direct adversarial self-play often suffers from cold-start dynamics when the initial policy lacks the specialized strategic depth required for exploring meaningful interactions (Vinyals et al., 2019; Silver et al., 2017). To provide stable initialization, we begin with imitation learning to distill the abilities embedded in expert trajectories from the teacher model. We collect two categories of trajectories using the teacher: adversarial episodes (agent com-pletes tasks in adversarial settings with attackers) and clean episodes (agent completes tasks without attacks). Including clean episodes prevents the stu-dent from becoming overly defensive. From these, we curate agent trajectories Dagent (successful com-pletions from both settings) and attacker trajecto-ries Dattacker (successful attacks). Following Wang et al. (2025c), we fine-tune the student with a KL-regularized SFT objective using LoRA (Hu et al., 2022):

5 Experiments 

In this section, we systematically evaluate DMAST to assess its efficacy in fortifying web agents against adversarial manipulations. Our experi-ments are designed to investigate: (i) the com-parative performance against other defense meth-ods; (ii) the co-evolutionary dynamics between the agent and the attacker during self-play, and (iii) the marginal contribution of each training stage. 

5.1 Experimental Setup Models. We instantiate our framework using Gemma-3-12B-IT as the student agent and Gemma-3-27B-IT as the teacher model for trajectory syn-thesis. During the evaluation phase, the 27B model serves as a fixed, high-capacity adversary to pro-vide a rigorous and consistent benchmark for de-fense performance. 

Training Data Synthesis. All training phases are conducted using the tasks in the MiniWob++ benchmark (Liu et al., 2018), excluding a fixed set of 28 tasks reserved for evaluation. Similar with Section 3.2,we augment the agentŌĆÖs prompt with synthetic sensitive information (e.g., passwords) and define information leakage via string-matching heuristics on generated tool parameters. For supervised initialization (Imitation Learn-ing and Oracle-Guided SFT), we utilize the full training task set. For the RL phase, we select only tasks where both the agent and attacker exhibit non-trivial success rates (agent > 0.2 in benign settings; attacker > 0.1 in adversarial settings), ensuring a dense optimization signal for stable co-evolutionary learning.

6 Conclusion 

We presented DMAST, a framework for harden-ing multimodal web agents against coordinated attacks across visual and textual observation chan-nels. By formalizing the agentŌĆōattacker interaction as a zero-sum Markov game, we first identified vi-sual injections as a particularly potent threat vector compared to text-only attacks. Our empirical anal-ysis demonstrates that each phase of the proposed training pipeline provides synergistic gains, with the full framework consistently outperforming ex-isting baselines on challenging out-of-distribution tasks. Specifically, DMAST establishes a superior balance between adversarial robustness and func-tional utility, a capability further validated by the emergence of a genuine co-evolutionary arms race during self-play. We hope our findings provide a practical foundation for building safer multimodal AI systems and motivate future research on scaling these defenses to broader attack objectives. 87 Limitations 

While DMAST establishes a robust foundation for dual-modality multi-stage adversarial safety train-ing in web agents, we identify several limitations that provide fertile ground for future investigation. 

Scope of Adversarial Objectives. Our current experiments primarily address sensitive data leak-age via prompt injection, a critical security bottle-neck in autonomous agents. However, the threat landscape is multifaceted, encompassing others such as control-flow hijacking and the dissemi-nation of misinformation. It is important to note that because DMAST is architected as a general-purpose Markov game, extending the defense to these alternative objectives involves only reconfig-uring the reward function R rather than modifying the underlying training paradigm. Future work could evaluate the performance of our method on other types of adversarial objectives. 

Absolute Capability of Small Models. Al-though our pipeline yields substantial relative im-provements, the absolute task success rate (10.2% on VisualWebArena) remains modest, largely due to the limited capacity of the 12B student model chosen for computational tractability. Applying DMAST to larger VLMs with stronger base capa-bilities is an important direction for future work. 

Potential Risks and Misuse Like other adversar-ial robustness research, our framework studies at-tack strategies in order to improve defenses. While such techniques could potentially be misused to de-sign more effective attacks, we believe that the pa-per does not introduce new real-world attack tools or deployment systems, and the primary benefit of this work is enabling safer deployment of multi-modal web agents by improving their robustness against cross-modal prompt injection.

Stage 2: Oracle-Guided SFT Data. The en-hanced SFT data is constructed via a four-stage pipeline (Section 4.2), also using the teacher model (Gemma-3-27B-IT): ŌĆó Golden trajectory collection: We first col-lect 4,000 clean episodes (no attacker) using the teacher agent. Only successful episodes are kept, and the agentŌĆÖs messages, observa-tions (screenshot + accessibility tree), and re-sponses are stored for each step. ŌĆó Attack synthesis: For each step in the golden trajectories, the teacher attacker generates HTML injection attacks to produce attacked versions of the observation (modified screen-shot and accessibility tree). This yields 4,000 attacked episodes that share the same ground-truth actions as the clean originals. ŌĆó Oracle CoT generation: An oracle model generates task-focused Chain-of-Thought (CoT) reasoning for each attacked step. The oracle has access to both the clean and at-tacked observations and produces a reasoning 12 trace that adheres to the zero-acknowledgment principle. ŌĆó Data mixing: We replace the original agentŌĆÖs CoT reasoning trace with the oracle-generated CoT in the attacked episodes. Then, for the dataset of clean episodes (i.e., agent completes tasks without attacks), to ensure diversity of the training data, instead of directly use the golden trajectories as agent successful trajec-tories, we collect another 4000 clean episodes without attacks and filter out the agent transi-tions from the agent successful episodes. ŌĆó Attacker transitions (2,997): from adversar-ial episodes where the attacker successfully induces information leakage. In total, each of the two SFT stages trains on a mixture of adversarial and clean data, balanced to ensure the agent acquires both attack robustness and standard task-completion ability.
