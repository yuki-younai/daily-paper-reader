## 摘要
层级指令遵循 (HIF) 指的是使用具有优先级排序的指令栈来提示大语言模型的问题。RLHF 和 DPO 等标准方法通常在此问题上失效，因为它们主要针对单一目标进行优化，未能显式地强制执行对系统提示词 (system prompt) 的合规性。同时，监督微调依赖于模仿经过过滤的合规数据，这无法在算法层面建立优先级的不对称性。在本文中，我们介绍了 HIPO，这是一种将 HIF 建模为受限马尔可夫决策过程 (Constrained Markov Decision Process) 的新型对齐框架。HIPO 将系统提示词从单纯的输入上下文提升为严格的算法边界。该算法采用原对偶安全强化学习方法，将系统提示词合规性作为显式约束进行动态强制执行，在这一可行域内严格最大化用户效用。对多种模型架构（如 Qwen、Phi、Llama）的广泛评估表明，HIPO 显著提高了系统合规性和用户效用。此外，机理分析表明，这种受限优化自主地驱动模型将其注意力转向长距离的系统标记 (system tokens)，为复杂工作流中可靠的大语言模型部署提供了理论基础。

## Abstract
Hierarchical Instruction Following (HIF) refers to the problem of prompting large language models with a priority-ordered stack of instructions. Standard methods like RLHF and DPO typically fail in this problem since they mainly optimize for a single objective, failing to explicitly enforce system prompt compliance. Meanwhile, supervised fine-tuning relies on mimicking filtered, compliant data, which fails to establish the priority asymmetry at the algorithmic level. In this paper, we introduce \textsc{HIPO}, a novel alignment framework that formulates HIF as a Constrained Markov Decision Process. \textsc{HIPO} elevates system prompts from mere input context to strict algorithmic boundaries. Using a primal-dual safe reinforcement learning approach, the algorithm dynamically enforces system prompt compliance as an explicit constraint, maximizing user utility strictly within this feasible region. Extensive evaluations across diverse model architectures (e.g., Qwen, Phi, Llama) demonstrate that \textsc{HIPO} significantly improves both system compliance and user utility. Furthermore, mechanistic analysis reveals that this constrained optimization autonomously drives the model to shift its attention toward long-range system tokens, providing a principled foundation for reliable LLM deployment in complex workflows.

在大型语言模型（LLM）的实际应用中，指令通常具有层级结构：**系统提示词（System Prompt）**设定全局规则、安全边界或角色，而**用户指令（User Prompt）**指定具体任务。然而，两者经常发生冲突（如用户诱导模型违反系统设定的安全规则）。
*   **研究动机**：现有的对齐方法（如 RLHF、DPO）通常将所有指令合并为单一目标进行优化，无法体现系统提示词的绝对优先级。监督微调（SFT）仅靠模仿合规数据，难以在算法层面建立“优先级不对称性”。
*   **核心问题**：如何在保证系统提示词合规性（硬约束）的前提下，最大化用户指令的执行效用（目标函数）。

#### 2. 论文提出的方法论：HIPO
论文提出了 **HIPO (Hierarchical Instruction Policy Optimization)** 框架，其核心思想是将层级指令遵循建模为**受限马尔可夫决策过程 (CMDP)**。
*   **核心思想**：将系统合规性视为约束条件，将用户效用视为优化目标。即：在满足“系统合规得分 $\ge \tau$”的约束下，最大化“用户效用得分”。
*   **关键技术细节**：
    *   **解耦奖励评估**：使用“LLM-as-Judge”机制，分别针对系统合规性（$r_{sys}$）和用户效用（$r_{user}$）进行独立打分，避免单一得分导致的信号干扰。
    *   **原始-对偶优化 (Primal-Dual)**：引入拉格朗日乘子 $\lambda$，将受限问题转化为无约束的极小极大问题。通过梯度上升更新策略参数 $\theta$，通过梯度下降动态调整 $\lambda$。当合规性不足时，$\lambda$ 增大以强化惩罚。
    *   **组相对策略优化 (GRPO)**：借鉴 DeepSeek-V3 的 GRPO 算法，通过组内样本的相对得分估计优势函数（Advantage），无需额外的 Critic 网络，降低了显存开销并提高了训练稳定性。

*   **性能领先**：HIPO 在所有测试模型上均实现了最佳的权衡，即在满足系统合规阈值的同时，用户效用显著高于 SFT 和 DPO。

*   **安全风险**：论文指出，如果系统提示词本身被恶意篡改，HIPO 强大的遵循能力可能会被利用来强制生成有害内容。
