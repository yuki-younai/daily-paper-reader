## 摘要
循环语言模型 (LoopLMs) 通过执行迭代潜层计算来优化内部表示，为显式思维链 (CoT) 推理提供了一种极具前景的替代方案。然而，现有的强化学习 (RL) 范式主要针对输出 token，这与推理过程隐式展开的循环架构在结构上存在不匹配。在这项工作中，我们提出了 LoopRPT，这是一个专为 LoopLMs 定制的强化预训练框架。通过将下一 token 预测重新定义为下一 token 推理任务，LoopRPT 利用 EMA 教师参考和噪声潜层展开，将强化信号直接分配给潜层步骤。这种形式化方法使 RL 能够直接塑造中间表示，从而将有效的推理压缩到更少的迭代次数中。我们在多种模型规模的 Ouro 架构上实例化了 LoopRPT。结果表明，LoopRPT 持续提升了每步表示的质量，在准确率与计算量的权衡中实现了帕累托占优。值得注意的是，在困难 token 上的显著提升表明 LoopRPT 增强了早期推理，而不仅仅是鼓励提前退出。我们的研究结果强调，强化预训练是学习 LoopLMs 中高效潜层推理的一种原则性范式。

## Abstract
Looped language models (LoopLMs) perform iterative latent computation to refine internal representations, offering a promising alternative to explicit chain-of-thought (CoT) reasoning. However, existing reinforcement learning (RL) paradigms primarily target output tokens, creating a structural mismatch with looped architectures whose reasoning unfolds implicitly. In this work, we propose LoopRPT, a reinforcement pre-training framework tailored for LoopLMs. By reframing next-token prediction as a next-token reasoning task, LoopRPT assigns reinforcement signals directly to latent steps using an EMA teacher reference and noisy latent rollouts. This formulation enables RL to directly shape intermediate representations, compressing effective reasoning into fewer iterations. We instantiate LoopRPT on the Ouro architecture across multiple model scales. Results demonstrate that LoopRPT consistently improves per-step representation quality, achieving Pareto dominance in accuracy-computation trade-offs. Notably, significant gains on hard tokens indicate that LoopRPT enhances early-stage reasoning rather than merely encouraging premature exits. Our findings highlight reinforcement pre-training as a principled paradigm for learning efficient latent reasoning in LoopLMs.

### 1. 核心问题与整体含义（研究动机和背景）
*   **研究动机**：现代大语言模型（LLM）主要通过显式的“思维链”（CoT）生成 token 来进行推理，这虽然有效但计算成本高且速度慢。**循环语言模型（LoopLMs）**（如 Ouro 架构）通过在潜空间（Latent Space）进行参数共享的迭代计算来优化内部表征，从而实现“隐式推理”，无需消耗额外的 token。
*   **核心问题**：现有的强化学习（RL）范式（如 RLVR）主要针对最终输出的 token 施加奖励，这与 LoopLMs 隐式展开的推理过程存在**结构性不匹配**。稀疏的 token 奖励难以有效指导潜空间的中间推理步骤，导致模型难以学习到高效的提前退出（Early-exit）策略。
*   **整体含义**：本文提出了 **LoopRPT** 框架，首次将强化预训练（RPT）引入循环架构，旨在通过密集的步骤级奖励信号，提升 LoopLMs 的推理质量并优化计算效率。

### 2. 方法论：核心思想与关键技术
LoopRPT 的核心思想是将“下一 token 预测”重构为“下一 token 推理”任务，直接在潜层迭代步骤中施加强化信号。
*   **基于熵的困难 Token 选择**：并非所有 token 都需要复杂推理。LoopRPT 使用教师模型的输出熵来识别“困难 token”，仅在这些关键位置应用 RL 更新，以提高学习效率。
*   **EMA 教师参考的步骤级奖励**：
    *   维护一个**指数移动平均（EMA）教师模型**作为稳定的基准。
    *   **准确率增益（Accuracy Gain）**：计算学生模型在每个迭代步骤的预测对数概率与教师参考步骤之间的差值。
    *   **难度感知的时间惩罚**：引入惩罚项以鼓励模型尽早退出，且惩罚力度根据 token 难度动态调整（简单 token 惩罚大，困难 token 允许更多计算）。
*   **噪声潜层回溯（Noisy Latent Rollouts）**：在循环隐藏状态中注入高斯噪声，通过多次采样（Rollouts）获得策略梯度，从而联合优化退出策略（Exit Policy）和骨干表征。
*   **联合优化目标**：结合了策略梯度损失（LPG）、步骤加权的表征学习损失（Lrep）、熵正则化和 KL 散度约束。

### 3. 实验设计
*   **模型架构**：在 **Ouro-1.4B** 和 **Ouro-2.6B** 架构上进行实例化。
*   **训练数据**：使用 **Omni-MATH**（包含约 4,428 个竞赛级数学问题及解答）。
*   **对比方法（Baselines）**：
    *   **Vanilla LLMs**：Qwen2.5/3、Llama3.1/3.2、Gemma3（涵盖 1B 到 12B 规模）。
    *   **原始 Ouro 模型**：未经过 LoopRPT 优化的循环模型。
    *   **CoT 模式**：对比标准模型在显式生成 CoT 时的表现。
*   **评估基准**：
    *   **下一 token 推理（NTR）**：评估在不同难度下的预测准确率与平均计算步数。
    *   **下游任务**：MMLU、MMLU-Pro、BBH、GSM8K（数学）、HumanEval/MBPP（代码）等。

### 4. 资源与算力
*   **硬件环境**：使用了 **8 台 NVIDIA A100 (80GB) GPU**。
*   **训练时长**：
    *   Ouro-1.4B 训练约需 **2 小时**。
    *   Ouro-2.6B 训练约需 **4 小时**。
*   **超参数**：最大序列长度 4096，Batch Size 为 8，训练 3 个 Epoch。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了从 1.4B 到 2.6B 的模型缩放实验，并在 10 多个主流基准测试集上进行了验证。
*   **消融实验**：非常充分。作者分别验证了高斯噪声、策略梯度（LPG）、表征学习（Lrep）、困难 token 选择以及时间惩罚项对最终性能的影响。
*   **客观性**：通过“帕累托曲线”（Pareto Curves）展示准确率与计算量的权衡，证明了 LoopRPT 在相同计算量下性能更优，或在相同性能下计算量更小，对比维度科学客观。

### 6. 主要结论与发现
*   **帕累托占优**：LoopRPT 在所有规模上均实现了准确率-计算量的最优平衡。例如，在 Ouro-2.6B 上，将困难 token 的平均步数从 3.51 降至 2.28，同时准确率提升了 3.58%。
*   **增强早期推理**：实验发现 LoopRPT 不仅仅是让模型“提前退出”，而是显著提升了前几个迭代步骤的表征质量（Step-1/2 的准确率大幅提升）。
*   **下游迁移能力**：在数学（GSM8K +3.60）和代码（MBPP+ +2.91）任务上表现出显著的性能增益，证明了预训练阶段学习到的潜层推理能力可以泛化。

### 7. 优点（亮点）
*   **理论完备性**：为循环架构的强化学习提供了原则性的框架，解决了潜层推理与稀疏奖励不匹配的问题。
*   **效率极高**：相比于显式 CoT，LoopRPT 在不增加推理延迟的前提下，通过优化潜层计算路径提升了模型上限。
*   **动态校准**：引入难度感知的时间惩罚，使模型能够根据 token 难度自适应分配计算资源。

### 8. 不足与局限
*   **架构依赖**：该方法目前高度依赖于 Ouro 这种特定的循环参数共享架构，在标准 Transformer 上的适用性尚未验证。
*   **数据偏差风险**：训练主要集中在数学竞赛数据（Omni-MATH）上，虽然在通用任务上有提升，但在非推理类（如创意写作、多轮对话）任务上的表现讨论较少。
*   **训练复杂性**：引入了 EMA 教师、多次噪声回溯等机制，相比标准预训练，其训练流程和超参数调优更为复杂。
