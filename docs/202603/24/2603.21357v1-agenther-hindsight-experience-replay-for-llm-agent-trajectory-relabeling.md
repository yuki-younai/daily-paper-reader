## 摘要
大语言模型（LLM）智能体在大多数现实任务中都会失败——GPT-4o 在 WebArena 导航任务中的成功率不足 15%，在 ToolBench 上的 pass@1 低于 55%（Zhou 等，2024；Qin 等，2024）——然而，每一个失败的轨迹通常都会被丢弃，从而浪费了所收集经验的主要来源。我们提出了 AgentHER，这是一个通过将事后经验重放（HER；Andrychowicz 等，2017）原理应用于自然语言智能体轨迹以进行离线数据增强，从而恢复这些丢失的训练信号的框架。其核心见解很简单：一个未能实现目标 A 的轨迹，往往是实现某个可达到的替代目标 B 的正确演示。AgentHER 通过一个四阶段流水线实现这一想法——失败分类、结果提取、带有置信度门控的 LLM 引导提示词重标记以及数据打包——将丢弃的失败案例转化为高质量的 SFT、DPO 和 ShareGPT 训练数据，并提供零成本的基于规则和基于 LLM 评判的两种实现方式。在 WebArena（Zhou 等，2024）和 ToolBench（Qin 等，2024）上，AgentHER 在四个模型系列（GPT-4o、Qwen2.5-72B/7B、LLaMA-3.1-8B）中，相比仅使用成功案例的 SFT 提升了 7.1-11.7 个百分点，同时实现了 2 倍的数据效率——仅需 50% 的成功演示即可达到基线性能。在 1.5B 到 72B 参数规模下，收益保持一致（+5.8-9.2 个百分点），并且在迭代重新部署下具有复合效应（在额外轮次中提升 +2.1 个百分点）。人工评估证实，在多评判员验证下，重标记的准确率达到了 97.7%。

## Abstract
LLM agents fail on the majority of real-world tasks -- GPT-4o succeeds on fewer than 15% of WebArena navigation tasks and below 55% pass@1 on ToolBench (Zhou et al., 2024; Qin et al., 2024) -- yet every failed trajectory is routinely discarded, wasting the dominant source of collected experience. We introduce AgentHER, a framework that recovers this lost training signal by adapting the Hindsight Experience Replay (HER; Andrychowicz et al., 2017) principle to natural-language agent trajectories for offline data augmentation. The key insight is simple: a trajectory that fails goal A is often a correct demonstration for some achievable alternative goal B. AgentHER realises this idea through a four-stage pipeline -- failure classification, outcome extraction, LLM-guided prompt relabeling with confidence gating, and data packaging -- that converts discarded failures into high-quality SFT, DPO, and ShareGPT training data, with both zero-cost rule-based and LLM-judge implementations. On WebArena (Zhou et al., 2024) and ToolBench (Qin et al., 2024), AgentHER improves over success-only SFT by +7.1-11.7 pp across four model families (GPT-4o, Qwen2.5-72B/7B, LLaMA-3.1-8B), while achieving 2x data efficiency -- matching baseline performance with only 50% of successful demonstrations. Gains are consistent from 1.5B to 72B parameters (+5.8-9.2 pp) and compound under iterative redeployment (+2.1 pp over additional rounds). Human evaluation confirms 97.7% relabeling precision under multi-judge verification.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：当前的 LLM 智能体在处理现实世界任务（如 WebArena 导航、ToolBench API 调用）时失败率极高（通常在 45%-85% 之间）。在传统的训练流程中，这些失败的轨迹（Trajectories）会被直接丢弃，导致了巨大的数据浪费。
*   **研究动机**：作者认为失败的轨迹并非毫无价值，它们往往包含了连贯的执行过程和正确的中间结果，只是未能达成最初设定的目标。
*   **整体含义**：借鉴强化学习中的**事后经验重放（Hindsight Experience Replay, HER）**思想，将“失败”转化为“成功”——即通过修改目标（Prompt），使原本失败的轨迹成为新目标的正确演示，从而大幅扩充高质量训练数据集。

### 2. 方法论：AgentHER 框架
AgentHER 包含一个四阶段的离线处理流水线：
1.  **失败检测（Failure Detector）**：对失败轨迹进行分类（如：结果错误、工具错误、幻觉等），并评估其“可恢复性”和“严重程度权重 ($w$)”。严重推理缺陷的轨迹会被剔除。
2.  **结果提取（Outcome Extractor）**：提取智能体实际达成的客观事实、数据或操作结果，作为重标记的依据。
3.  **提示词重标记（Prompt Relabeler）**：利用 LLM 根据实际结果合成一个新的、自然的、且该轨迹能完美达成的用户提示词 ($\hat{g}$)。
    *   **关键技术**：引入了**多评判员验证（Multi-judge Verification）**和**置信度门控（Confidence Gating）**，只有当两个独立的 LLM 评判员都给出高分时，重标记才被接受。
4.  **数据打包（Data Augmenter）**：将验证后的（新目标, 轨迹）对转化为 SFT（监督微调）、DPO（直接偏好优化）或 ShareGPT 格式。
    *   **DPO 创新**：在 DPO 中保持轨迹固定，对比“新目标（正例）”和“原目标（负例）”，并使用严重程度权重 $w$ 缩放损失函数。

*   **Benchmark**：以官方测试集的成功率（Success Rate）和 Pass@1 为主要指标。

### 7. 优点

*   **计算开销**：虽然节省了环境交互，但重标记过程（尤其是多轮尝试和多评判员）仍需消耗一定的 API Token 或推理算力。
