## 摘要
虽然大语言模型（LLMs）推动了通用智能体的发展，但在未见任务上实现鲁棒的泛化仍然是一个重大挑战。目前的方法通常依赖于微调或利用检索经验的免训练记忆增强生成；然而两者都有局限性：微调往往难以外推到新任务，而经验检索的表现通常不如监督基准。在这项工作中，我们建议结合这些方法，并系统地研究如何训练检索增强型 LLM 智能体，以有效地在上下文中利用检索到的轨迹。首先，我们利用 LoRA 建立了一套鲁棒的有监督微调（SFT）方案，其性能优于几种最先进的智能体训练流水线。其次，我们对经验检索的关键设计选择进行了详细分析，确定了存储、查询和轨迹选择的最佳策略。最后，我们提出了一种将经验检索集成到微调过程中的流水线。我们的结果表明，这种结合方法显著提高了对未见任务的泛化能力，为构建能够从经验中学习的智能体提供了一个可扩展且有效的框架。

## Abstract
While large language models (LLMs) have advanced the development of general-purpose agents, achieving robust generalization to unseen tasks remains a significant challenge. Current approaches typically rely on either fine-tuning or training-free memory-augmented generation using retrieved experience; yet both have limitations: fine-tuning often fails to extrapolate to new tasks, while experience retrieval often underperforms compared to supervised baselines. In this work, we propose to combine these approaches and systematically study how to train retrieval-augmented LLM agents to effectively leverage retrieved trajectories in-context. First, we establish a robust supervised fine-tuning (SFT) recipe using LoRA that outperforms several state-of-the-art agent training pipelines. Second, we provide a detailed analysis of key design choices for experience retrieval, identifying optimal strategies for storage, querying, and trajectory selection. Finally, we propose a pipeline that integrates experience retrieval into the fine-tuning process. Our results demonstrate that this combined approach significantly improves generalization to unseen tasks, providing a scalable and effective framework for building agents that learn to learn from experience.

### 2. 方法论：ExpRAG 与 ExpRAG-LoRA
论文提出了 **ExpRAG（Experience Retrieval-Augmented Generation）** 框架：
*   **核心思想**：将智能体过去的交互轨迹（Trajectories）视为“经验”，在决策时检索最相关的经验并将其注入系统提示词中。
*   **关键技术细节**：
    1.  **经验银行构建**：离线收集智能体或专家的成功/失败轨迹，将其序列化为多轮对话格式，并使用向量模型（如 Qwen3-Embedding）生成嵌入索引。
    2.  **检索机制**：
        *   **静态检索**：在任务开始时（$t=0$）根据任务描述检索一次。
        *   **动态检索**：在每个决策步（$t$）根据当前交互历史实时刷新检索结果。
    3.  **内存块注入**：将检索到的 Top-K 条轨迹格式化为 JSON 或文本，区分成功与失败案例，插入 LLM 的系统提示词。
    4.  **ExpRAG-LoRA（核心创新）**：在微调阶段就引入检索到的经验。通过在训练时让模型接触带有相关经验的上下文，训练模型“利用参考案例解决当前问题”的能力，而不仅仅是死记硬背动作序列。

### 6. 主要结论与发现
*   **ExpRAG-LoRA 显著提升泛化性**：在未见过的“困难”任务上，ExpRAG-LoRA 的表现远超标准 LoRA。例如在 ALFWorld 困难任务上，Ministral 3-8B 从 34.4% 提升至 88.5%。
*   **延迟泛化现象**：研究发现智能体微调中存在类似“Grokking”的现象——即使验证集损失（Loss）开始上升，模型在未见任务上的成功率仍可能继续提高。
*   **简单即有效**：简单的、只读的完整轨迹检索（ExpRAG）在很多情况下优于复杂的、带有自我反思或压缩机制的记忆系统。
*   **模型容量影响**：长文本模型（如 Qwen 2.5-7B-1M）能更好地从更多的检索经验（更大的 K 值）中获益。

### 7. 优点与亮点
*   **建立了强基准**：论文指出现有文献中 SFT 基准往往偏弱，通过优化 LoRA 方案，作者建立了一个极具竞争力的 Baseline。
*   **系统性分析**：对 RAG 在智能体领域的应用做了非常细致的工程化探索（如 JSON 格式的影响、动态检索的稳定性问题）。
*   **训练与检索的结合**：证明了“在微调中加入检索”是让模型学会利用上下文经验的关键。

### 8. 不足与局限
*   **依赖高质量经验**：实验中使用的经验银行主要来自专家脚本，未充分探讨当经验库包含大量 LLM 自身生成的低质量/噪声轨迹时，系统的鲁棒性如何。
*   **静态记忆限制**：采用的是固定的、只读的经验库，缺乏在线更新、记忆压缩或遗忘机制，对于极长期的持续学习可能存在瓶颈。
*   **推理开销**：随着检索轨迹数量（K）增加，Prompt 长度线性增长，虽然成功率提高，但也显著增加了推理延迟和显存占用。
