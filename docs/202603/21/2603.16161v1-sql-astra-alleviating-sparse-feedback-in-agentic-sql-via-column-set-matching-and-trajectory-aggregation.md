## 摘要
智能体强化学习 (Agentic RL) 在复杂任务中展现出应用前景，但 Text-to-SQL 仍主要局限于单轮范式。一个主要的瓶颈是信用分配 (credit assignment) 问题。在传统范式中，奖励仅由最后一轮的反馈决定，忽略了中间过程，导致信用评估不明确。为此，我们提出了 Agentic SQL，这是一个具有通用两层奖励机制的框架，旨在提供有效的轨迹级评估和密集的步骤级信号。首先，我们引入了聚合轨迹奖励 (ATR) 来解决多轮信用分配。通过使用非对称转移矩阵，ATR 聚合了面向过程的分数，以激励持续改进。利用李雅普诺夫 (Lyapunov) 稳定性理论，我们证明了 ATR 充当能量耗散算子，保证了无环策略和单调收敛。其次，列集匹配奖励 (CSMR) 提供即时的步骤级奖励以缓解稀疏性。通过在每一轮执行查询，CSMR 根据部分正确性将二进制 (0/1) 反馈转换为密集的 [0, 1] 信号。在 BIRD 上的评估显示，相比于二进制奖励的 GRPO，性能提升了 5%。值得注意的是，在使用相同模型的情况下，我们的方法在 BIRD 和 Spider 2.0 上优于 SOTA 的 Arctic-Text2SQL-R1-7B，推动了 Text-to-SQL 向鲁棒的多轮智能体范式发展。

## Abstract
Agentic Reinforcement Learning (RL) shows promise for complex tasks, but Text-to-SQL remains mostly restricted to single-turn paradigms. A primary bottleneck is the credit assignment problem. In traditional paradigms, rewards are determined solely by the final-turn feedback, which ignores the intermediate process and leads to ambiguous credit evaluation. To address this, we propose Agentic SQL, a framework featuring a universal two-tiered reward mechanism designed to provide effective trajectory-level evaluation and dense step-level signals. First, we introduce Aggregated Trajectory Reward (ATR) to resolve multi-turn credit assignment. Using an asymmetric transition matrix, ATR aggregates process-oriented scores to incentivize continuous improvement. Leveraging Lyapunov stability theory, we prove ATR acts as an energy dissipation operator, guaranteeing a cycle-free policy and monotonic convergence. Second, Column-Set Matching Reward (CSMR) provides immediate step-level rewards to mitigate sparsity. By executing queries at each turn, CSMR converts binary (0/1) feedback into dense [0, 1] signals based on partial correctness. Evaluations on BIRD show a 5% gain over binary-reward GRPO. Notably, our approach outperforms SOTA Arctic-Text2SQL-R1-7B on BIRD and Spider 2.0 using identical models, propelling Text-to-SQL toward a robust multi-turn agent paradigm.

### SQL-ASTRA 论文深度总结

#### 1. 核心问题与整体含义（研究动机和背景）
传统的 Text-to-SQL 任务大多局限于**单轮生成范式**，这与人类数据分析师通过多次尝试、观察结果并修正策略的真实过程不符。在引入强化学习（RL）优化 SQL 生成时，面临三大挑战：
*   **范式受限**：现有方法缺乏动态交互，难以处理复杂的现实问题。
*   **信用分配困难**：多轮交互中通常只根据最终结果给出反馈（全或无），导致模型无法识别中间哪些步骤对成功有贡献。
*   **奖励信号稀疏**：即使有步骤级反馈，通常也是二进制（0/1）的执行成功信号，忽略了“部分正确”查询中的丰富信息。

**SQL-ASTRA** 的提出旨在通过构建一个多轮交互的 Agent 框架，并设计一套双层奖励机制，来缓解反馈稀疏问题并实现精准的信用分配。

#### 2. 方法论：核心思想与关键技术
该论文提出了 **Agentic SQL** 框架，其核心由两个关键奖励组件构成：

*   **列集匹配奖励 (CSMR - Column-Set Matching Reward)**：
    *   **核心思想**：将传统的行级精确匹配放宽为“列值集合”的匹配。
    *   **技术细节**：提取预测结果和真实结果中每一列的唯一值集合，计算匹配的列数比例。为了区分“伪完美匹配”（列值对但行组合错）和真正的完美匹配，引入缩放因子 $\alpha$（如 0.8），仅给真正的完美匹配 1.0 分，其余部分正确的情况给出 $[0, \alpha]$ 之间的密集分数。
*   **聚合轨迹奖励 (ATR - Aggregated Trajectory Reward)**：
    *   **核心思想**：解决多轮交互中的信用分配，激励模型持续改进。
    *   **技术细节**：使用一个**非对称转移矩阵 $M$** 来评估状态迁移。如果当前轮次比上一轮有进步（奖励增加），给予正向激励；如果退步，则给予更重的惩罚（非对称性：惩罚力度大于奖励力度）。
    *   **理论支撑**：引入**李雅普诺夫（Lyapunov）稳定性理论**，将 SQL 纠错过程建模为动力系统。证明了 ATR 充当能量耗散算子，从数学上保证了策略不会陷入死循环（无环性）且能单调收敛至正确答案。
*   **训练算法**：采用 **GRPO**（群组相对策略优化）算法，并引入 **Tool Masking** 技术，使模型只针对推理 Token 计算损失，而不学习执行结果 Token。
