## 摘要
大语言模型（LLMs）展现出强大的推理能力，但通常需要昂贵的训练后处理才能达到高性能。最近的测试时对齐方法提供了一种轻量级的替代方案，但目前主要针对偏好对齐而非推理进行了探索。为了填补这一空白，我们提出了标记级自适应路由（TARo），它完全在推理阶段引导冻结的大语言模型进行结构化推理。具体而言，我们首先在分步数学轨迹上训练奖励模型，以捕捉细粒度的逻辑一致性信号，然后引入一个可学习的标记级路由器，自动控制奖励模型对基座模型的引导。广泛的实验表明，TARo 显著提升了推理性能，相比基座模型提升高达 +22.4%，相比现有的标记级测试时对齐方法提升 +8.4%，同时还增强了分布外临床推理（MedXpertQA）和指令遵循（AlpacaEval）能力。此外，TARo 还能在无需重新训练的情况下从小型骨干网络泛化到大型骨干网络，将测试时对齐从偏好优化扩展到了鲁棒的跨领域推理。

## Abstract
Large language models (LLMs) exhibit strong reasoning capabilities but typically require expensive post-training to reach high performance. Recent test-time alignment methods offer a lightweight alternative, but have been explored mainly for preference alignment rather than reasoning. To bridge this gap, we propose, Token-level Adaptive Routing (TARo), which steers frozen LLMs toward structured reasoning entirely at inference time. Specifically, we first train reward models on step-wise mathematical traces to capture fine-grained logical consistency signals, then introduce a learnable token-level router that automatically controls the guidance of the reward model to the base model. Extensive experiments show that TARo significantly improves reasoning performance by up to +22.4% over base model and +8.4% over existing token-level test-time alignment methods, while also boosting out-of-distribution clinical reasoning (MedXpertQA) and instruction following (AlpacaEval). Furthermore, TARo also generalizes from small to large backbones without retraining, extending test-time alignment from preference optimization to robust, cross-domain reasoning.

### 论文总结：TARo - 大语言模型测试时对齐的标记级自适应路由

#### 1. 核心问题与整体含义（研究动机和背景）
大语言模型（LLMs）在复杂推理任务中表现出色，但通常依赖高昂的训练后处理（如强化学习）。现有的“测试时对齐”（Test-time Alignment）技术虽然提供了一种无需重新训练基座模型的轻量级方案，但存在两大局限：
*   **缺乏推理针对性**：大多侧重于人类偏好对齐，而非严密的逻辑推理。
*   **引导机制僵化**：现有方法（如 GenARM）通常使用固定的插值权重来混合基座模型和奖励模型的输出，无法根据生成过程中的不同 Token 或不同领域动态调整引导强度。
**TARo** 的提出旨在通过**标记级（Token-level）的自适应路由**，在推理阶段动态引导冻结的 LLM 进行结构化推理，解决固定权重带来的鲁棒性差和跨领域失效问题。

#### 2. 方法论：核心思想与技术细节
TARo 的核心思想是将基座模型和奖励模型视为“专家”，通过一个轻量级的路由器在每个解码步骤动态决定两者的混合比例。
*   **推理奖励模型训练**：在分步数学推理轨迹数据集（Math-StepDPO-10K）上训练奖励模型。利用 DPO（直接偏好优化）损失函数，使模型能够识别逻辑正确的推理步骤。
*   **标记级自适应路由器**：
    *   **输入特征**：路由器接收基座模型和奖励模型在当前步骤产生的 Logits（概率分布）。研究探讨了“全 Logits 拼接”和“Top-k Logits + 索引嵌入”两种设计。
    *   **动态权重生成**：路由器（一个简单的 MLP）输出一个标量 $\alpha_t \in (0, 1)$，用于混合两个模型的 Logits：$z_{guided} = (1 - \alpha_t) z_{base} + \alpha_t z_{reward}$。
    *   **训练目标**：结合负对数似然（NLL）损失和熵正则化项。熵正则化鼓励路由器在模型分歧时做出“果断”的选择（即 $\alpha_t$ 趋向 0 或 1），而非模糊的平均。
*   **推理流程**：在解码时，基座模型和奖励模型同时进行前向计算，路由器实时调整混合比例，生成最终的 Token 分布。

*   **自适应性强**：路由器能自动识别哪些 Token 需要奖励模型引导（如数学算式、推理步骤词），哪些 Token 应由基座模型主导（如上下文描述词）。

#### 7. 优点：亮点与创新
*   **动态性**：打破了测试时对齐中固定权重的传统，实现了 Token 级别的细粒度控制。
*   **通用性**：虽然奖励模型是在数学数据上训练的，但通过自适应路由，它在医疗和通用任务中也表现出正向引导作用。
*   **非侵入性**：无需修改基座模型参数，非常适合闭源模型或超大规模模型的推理增强。
*   **可解释性**：通过分析 $\alpha_t$ 的分布，可以直观观察到模型在推理过程中的决策偏好。

#### 8. 不足与局限
*   **基座模型依赖**：在基座模型本身极强（如 Qwen-2.5-3B）且任务偏向通用指令时，提升幅度相对较小。

*   **错误修正能力**：定性分析显示，如果奖励模型和基座模型在推理逻辑上同时出错，TARo 缺乏自我纠错或回溯机制。
